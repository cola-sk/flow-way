# 摄像头检测 - 简化版（向量判断法）

## 🎯 核心概念：一句话说清楚

**摄像头会不会拍到你的车？只看一件事：你的行进方向 和 摄像头的拍摄方向，夹角是否在90度以内。**

---

## 📐 数学模型（超简单版）

### 第一步：计算路线方向角

```
起点 (39.90, 116.40)
  ↓
  计算方向角 = 东向 (90°)
  ↓
终点 (39.90, 116.50)
```

使用公式：根据两点坐标，算出行进方向是北东西南哪一个（0-360度）

### 第二步：获取摄像头方向角

```
摄像头名称："东向西（高峰期）"
  ↓
解析方向 = "东向西" = 270° (西方向)
```

### 第三步：算夹角

```
路线方向：90° (东)
摄像头方向：270° (西)
夹角 = |90 - 270| = 180°

180° > 90° → 不会被拍到 ✓
```

### 第四步：得出结论

```
如果 夹角 ≤ 90° → 会被拍到 ⚠️
如果 夹角 > 90°  → 不会被拍到 ✓
```

## 🗺️ 可视化示例

### 场景：十字路口摄像头

```
         北(0°)
          ↑
    西(270°)←→东(90°)
          ↓
        南(180°)

摄像头位置：中心 ⊙
摄像头拍摄方向：东向西 = 270° (指向西)
拍摄范围（容差±90°）：180° ~ 360°（西北到西南扇形区域）
```

### 四条路线的判断结果

#### 路线1：西→东（方向角90°）
```
起点 ← 摄像头 ⊙ → 终点

行进方向：东 (90°)
拍摄方向：西 (270°)
夹角：|90 - 270| = 180°

180° > 90° → ✓ 不会被拍到
（迎面而来，摄像头看不到你的方向）
```

#### 路线2：东→西（方向角270°）
```
起点 → 摄像头 ⊙ ← 终点

行进方向：西 (270°)
拍摄方向：西 (270°)
夹角：|270 - 270| = 0°

0° ≤ 90° → ⚠️ 会被拍到
（同方向行进，摄像头能拍到）
```

#### 路线3：南→北（方向角0°）
```
        终点↑
        摄像头⊙
        起点↓

行进方向：北 (0°)
拍摄方向：西 (270°)
夹角：|0 - 270| = 270° → 最小角 = 360-270 = 90°

90° ≤ 90° → ⚠️ 勉强会被拍到（边界情况）
```

#### 路线4：北→南（方向角180°）
```
        起点↑
        摄像头⊙
        终点↓

行进方向：南 (180°)
拍摄方向：西 (270°)
夹角：|180 - 270| = 90°

90° ≤ 90° → ⚠️ 勉强会被拍到（边向内）
```

## 🔧 代码实现

### 最简单的判断函数

```typescript
/**
 * 只需要这三个参数就能判断
 */
function isDetected(
  routeBearing: number,      // 路线方向（0-360）
  cameraBearing: number,     // 摄像头拍摄方向（0-360）
  tolerance: number = 90     // 容差角度，默认90度
): boolean {
  // 计算最小夹角
  let angle = Math.abs(routeBearing - cameraBearing);
  if (angle > 180) angle = 360 - angle;
  
  // 简单判断
  return angle <= tolerance;
}

// 使用
isDetected(90, 270);   // false (东 vs 西)
isDetected(270, 270);  // true  (西 vs 西)
isDetected(180, 270);  // true  (南 vs 西, 角度90)
```

## ⚙️ 参数调整

### 容差角度（detectionAngle）

容差角度决定了"夹角多大以内算被拍到"

```
detectionAngle = 45°
└─ 严格模式，只有方向非常接近才算被拍到
└─ 例：路线东(90°) vs 摄像头东北(45°) 的夹角45°，勉强被拍到

detectionAngle = 90° (推荐)
└─ 平衡模式，±90度范围内算被拍到
└─ 例：路线东(90°) vs 摄像头南(180°) 的夹角90°，勉强不被拍到

detectionAngle = 180°
└─ 宽松模式，所有方向都算被拍到
└─ 不建议，太过保守
```

## 🧪 测试用例

```typescript
// 测试1：同向行驶
assert(isDetected(270, 270) === true);
// 摄像头和你都往西走，肯定被拍到

// 测试2：对向行驶
assert(isDetected(90, 270) === false);
// 摄像头往西，你往东，迎面而来，拍不到

// 测试3：垂直十字
assert(isDetected(0, 270) === false);
// 摄像头往西，你往北，夹角90度，刚好不拍到

// 测试4：垂直十字边界
assert(isDetected(90, 270) === false);
// 夹角180度，绝对拍不到

// 测试5：接近方向
assert(isDetected(85, 90) === true);
// 摄像头往东，你也往东，方向接近，肯定被拍到
```

## 🎯 为什么这个方法比距离判断更准确

| 判断方式 | 缺点 | 优点 |
|---------|------|------|
| **距离判断** | 摄像头在100米内就算会被拍到，但实际方向不对也不会被拍 | 计算简单 |
| **方向判断** | 需要计算两个方向的夹角，稍微复杂一点 | **准确度高，真实反映摄像头能否拍到** |
| **方向+距离** | 计算量最大 | **最准确**（既考虑是否靠近，又考虑方向） |

## 🔑 总结

**你要记住的就是这个简单规则：**

```
夹角 = |路线方向 - 摄像头方向|
如果 夹角 > 180，用 360 - 夹角
如果 夹角 ≤ 90  → 会被拍到 ⚠️
如果 夹角 > 90   → 不会被拍到 ✓
```

就这么简单！🎉
